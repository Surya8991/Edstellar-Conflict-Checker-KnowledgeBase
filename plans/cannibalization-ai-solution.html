Keyword Cannibalization · Plan

How the AI turns each cannibalization conflict into a ready-to-apply fix

The tool already finds the URLs competing for the same search query. This plan covers the next step: the AI recommending the actual fix, with the on-page SEO changes written out for you to approve.

Plan only. Nothing is built or changed yet.

The problem, in one line: when two or more Edstellar URLs target the same keyword, Google splits ranking signals between them. That is keyword cannibalization, and it makes your own pages compete with each other, so both rank lower than one strong page would.

The idea in five simple steps

Step 3, up close

The three fixes the AI will propose

Every conflict fits one of three shapes. For each one, the AI hands you something you can act on straight away.

Consolidate

Merge and 301 redirect

When the URLs are near-duplicates targeting the same query with the same intent.

  • Picks the primary URL to keep
  • Lists the losing URLs to 301 redirect into it
  • Drafts a merged content outline that pulls the best sections from the pages being retired

Example: keep the strongest "corporate training companies" page, then 301 redirect the two weaker URLs into it.

Differentiate

Give each a distinct target keyword

When the pages overlap but both add value and should both stay live.

  • Assigns a distinct target keyword to each page
  • Rewrites the meta title and H1 so they stop overlapping
  • Names the search intent each page should own

Example: one page targets "for startups" and the other targets "for enterprises", so they no longer compete for the same query.

Monitor and protect

Nudge, do not merge

When different content types clash, for example a blog outranking a course page.

  • Suggests internal links with keyword-rich anchor text pointing to the page that should rank
  • Recommends de-optimizing the off-target page for that keyword
  • Never 301 redirects across different search intents

Example: link the popular blog to the course page so buyers land on the page that converts.
